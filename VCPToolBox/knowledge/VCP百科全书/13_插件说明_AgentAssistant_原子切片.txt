[VCP百科全书] 插件说明——AgentAssistant（女仆团协作）原子切片

一、插件身份与运行方式
- manifest 名称：`AgentAssistant`
- 显示名：`女仆团协作插件`
- 类型：`hybridservice`
- 通讯协议：`direct`
- 入口：`AgentAssistant.js`
- 版本：`2.0.0`
- 配置项：`AGENT_ASSISTANT_MAX_HISTORY_ROUNDS`、`AGENT_ASSISTANT_CONTEXT_TTL_HOURS`

二、核心能力（按 manifest + README）
1) Agent 间即时通讯
- 通过 `agent_name + prompt` 向指定女仆 Agent 发起咨询/协作。
- 目标 Agent 基于其知识域与上下文返回结果。

2) 会话上下文保持
- 插件按“会话”维护历史轮次，支持连续协作对话。
- 轮次上限与上下文保留时长由配置项控制。

3) 未来电话（定时通讯）
- 传入 `timely_contact` 可把即时请求转为未来任务。
- 时间格式：`YYYY-MM-DD-HH:mm`。
- 任务由主系统调度，创建后先返回“已预约”确认。

三、调用参数约束
- `tool_name`：必须为 `AgentAssistant`。
- `agent_name`：必填，需与系统配置中的中文 Agent 名匹配（如：小娜/小克/小吉/小冰/小雨/小绝/小芸）。
- `prompt`：必填，建议在开头带简短自我介绍，避免目标 Agent 无法判断咨询身份。
- `timely_contact`：可选，未来时间点发送。
- `temporary_contact`：示例中出现，可用于临时通讯场景（是否保留上下文需按实现版本确认）。

四、返回与行为特征
- 即时通讯：直接返回目标 Agent 的应答内容。
- 定时通讯：先返回预约成功回执；到时后由调度器执行并通知日志侧客户端。
- 该插件本质是“Agent 协作协议桥”，不是知识检索插件本体。

五、常见错误与修复
- 目标 Agent 不存在/名称不匹配
  - 原因：`agent_name` 与系统配置不一致。
  - 修复：使用配置中的准确中文名。
- 参数缺失
  - 原因：漏传 `agent_name` 或 `prompt`。
  - 修复：补齐必填字段并检查包裹符。
- 时间格式错误
  - 原因：`timely_contact` 非 `YYYY-MM-DD-HH:mm`。
  - 修复：按标准格式重写，避免自然语言时间。
- 协作回复质量不稳定
  - 原因：`prompt` 无身份信息、任务边界不清。
  - 修复：先自我介绍，再给目标、上下文、输出要求。

六、最佳实践（RAG原子文档）
- 选 Agent 先看专长，再发任务，避免“跨域硬问”。
- `prompt` 使用“三段式”：身份 -> 目标 -> 输出格式。
- 定时任务优先用于提醒/复盘/周期性事务，不用于强实时场景。
- 对关键任务记录“预约时间+目标Agent+预期产出”，便于追踪。
- 出现失败时保留原始参数，便于复现与二次召回。

七、调用示例
<<<[TOOL_REQUEST]>>>
tool_name:「始」AgentAssistant「末」,
command:「始」AskMaidAgent「末」,
agent_name:「始」小娜「末」,
prompt:「始」我是主Agent，请帮我总结今天的研发风险与优先级。「末」
<<<[END_TOOL_REQUEST]>>>
