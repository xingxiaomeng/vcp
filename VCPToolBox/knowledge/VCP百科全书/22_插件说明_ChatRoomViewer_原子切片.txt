[VCP百科全书] ChatRoomViewer (VCPChat客户端信息查看器) - 静态状态信息插件

一、插件基本信息
1. 插件名称：ChatRoomViewer / VCPChat客户端信息查看器
2. 插件类型：静态插件 (static)
3. 版本号：2.1.0
4. 所属模块：VCPDistributedServer/Plugin
5. 功能一句话总结：提供VCPChat客户端实时运行状态和配置信息

二、功能定位
ChatRoomViewer是一个静态数据注入插件，通过系统提示词占位符向AI提供VCPChat客户端的全面运行状态信息。它监控并收集主题设置、界面模式、窗口状态、Agent配置、会话时间流逝等多维度数据，使AI能够感知当前客户端环境，实现上下文自适应的交互体验。

适用场景：
- AI需要了解当前VCPChat客户端配置
- 根据主题自适应调整回复样式（如亮暗模式气泡）
- 获取会话历史和持续时间信息
- 查询已配置的Agent列表和属性

三、配置参数

配置文件位置：VCPDistributedServer/Plugin/ChatRoomViewer/config.env

参数列表：
1. DebugMode (boolean, 可选, 默认false)
   - 描述：是否启用详细调试日志输出到stderr
   - 默认值：true（config.env中配置）

2. Enabled (boolean, 可选, 默认true)
   - 描述：是否启用此插件
   - 默认值：true

3. VCPChatRoot (string, 可选, 默认空字符串)
   - 描述：VCPChat客户端根目录的绝对路径
   - 默认值：空（自动检测）

4. TimeZone (string, 可选, 默认Asia/Shanghai)
   - 描述：用于时间戳显示的时区
   - 默认值：Asia/Shanghai（东八区）

四、调用命令列表
本插件为静态插件，不提供主动调用命令。所有功能通过系统提示词占位符自动注入。

系统提示词占位符列表：

1. {{VCPChatStatus}}
   功能描述：提供当前VCPChat客户端的运行状态和配置信息，包括系统信息、节点信息、用户设置等
   注入内容示例：操作系统类型、VCPChat版本、当前工作目录、分布式节点连接状态等

2. {{VCPChatTheme}}
   功能描述：提供当前使用的主题和界面设置信息
   注入内容：主题名称、颜色模式(light/dark)、颜色配置、壁纸信息

3. {{VCPChatSessionWatcher}}
   功能描述：提供当前会话监控信息
   注入内容：监控的history.json文件路径、监控状态（启用/禁用）

4. {{VCPChatAgent}}
   功能描述：提供所有VCPChat Agent的基本信息
   注入内容：Agent名称列表、模型配置、头像文件地址

5. {{VCPChatModeBubbleTip}}
   功能描述：主题模式自适应气泡实现指导
   注入内容：教授Agent如何创建亮暗模式自动切换的气泡样式的CSS代码示例

6. {{VCPChatSessionTimeElapsed}}
   功能描述：提供当前会话时间流逝信息
   注入内容：距离上次对话的时间、会话持续时间、集中交流时段分析

7. {{VCPChatGroupSessionWatcher}}
   功能描述：提供群聊会话监控信息
   注入内容：当前监控的群聊会话文件地址和状态

五、使用示例

本插件无需主动调用，在系统提示词模板中使用占位符即可：

示例1：在系统提示词中获取客户端状态
```
你是VCPChat的智能助手。当前客户端状态如下：
{{VCPChatStatus}}
请根据以上信息为用户提供帮助。
```

示例2：获取主题信息以自适应样式
```
当前主题配置：
{{VCPChatTheme}}

请使用以下方式创建自适应气泡：
{{VCPChatModeBubbleTip}}
```

示例3：了解当前可用Agents
```
系统中配置了以下AI角色：
{{VCPChatAgent}}

你可以根据用户需求调用合适的Agent。
```

示例4：获取会话时间信息
```
会话信息：
{{VCPChatSessionTimeElapsed}}
{{VCPChatSessionWatcher}}
```

六、注意事项与限制

1. 刷新机制：
   - 刷新间隔：每秒刷新一次（Cron表达式：* * * * * *）
   - 数据通过chokidar文件监控器实时更新
   - 监控的文件包括：settings.json、themes.css、Agents目录下的config.json

2. 数据获取方式：
   - 读取VCPChat主目录下的AppData/settings.json
   - 读取styles/themes.css
   - 扫描AppData/Agents/目录下的各Agent配置
   - 监控会话history.json文件

3. 自动检测逻辑：
   - 如果VCPChatRoot未设置，插件会自动向上查找包含main.js的目录
   - 使用Node.js的require.main.path进行路径推断

4. 时区处理：
   - 支持自定义时区（默认Asia/Shanghai）
   - 时间戳格式化使用Intl.DateTimeFormat API
   - 支持24小时制显示

5. 性能考虑：
   - 使用文件缓存避免重复读取
   - 仅在文件变化时更新数据
   - 监控模式可通过--watch参数启用

6. 已知限制：
   - 依赖VCPChat的文件结构，如果结构改变可能导致数据获取失败
   - 群聊功能需要特定配置才能启用
   - 部分信息（如Agent详情）需要Agent已配置才能显示
