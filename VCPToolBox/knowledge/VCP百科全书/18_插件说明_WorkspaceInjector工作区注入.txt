今日系统整理了WorkspaceInjector（工作区动态注入）插件的完整使用说明，作为客服知识库。

核心概念：WorkspaceInjector是一个消息预处理插件（messagePreprocessor类型），通过在系统提示词中使用{{Workspace::别名或路径::深度}}占位符，将本地文件夹的完整目录结构动态注入到AI的上下文中，极大地帮助AI理解项目代码的全局结构。

插件基本信息：插件名称WorkspaceInjector，类型messagePreprocessor消息预处理插件，版本1.2.0，入口文件injector.js，通信协议direct。该插件在每次消息处理时自动运行，无需在系统提示词中添加工具说明占位符，直接在提示词中使用{{Workspace::...}}占位符即可触发。

配置项说明：WORKSPACE_ALIASES是工作区路径别名，JSON格式字符串，在config.env中配置，例如：WORKSPACE_ALIASES={"w": "D:\\VCPToolBox", "docs": "C:\\Users\\Name\\Documents"}。注意路径中的反斜杠必须转义为双反斜杠。

依赖安装：需要在VCPToolBox项目根目录下运行npm install ignore安装依赖，否则.workspaceignore忽略文件功能无法使用。

四种使用方式：

方式一，使用别名（适合固定项目）：在config.env中配置别名后，在系统提示词中写入{{Workspace::w}}，插件会自动将w替换为对应路径的完整目录树。

方式二，使用直接路径（适合临时任务）：直接在占位符中填入文件夹的绝对路径，如{{Workspace::D:\VCPToolBox\Agent}}，无需任何配置和重启，非常方便。

方式三，使用别名/子目录组合（最灵活）：将配置好的别名作为根目录，附加任意层级的子目录，如{{Workspace::w/Plugin}}会自动解析为别名对应路径下的Plugin子目录。支持多层子目录，如{{Workspace::w/Plugin/WorkspaceInjector}}。

方式四，限制目录深度（防止Token爆炸）：在占位符末尾添加::深度参数，如{{Workspace::w::2}}只显示到第二层级，{{Workspace::D:\VCPToolBox\Plugin::1}}只显示Plugin文件夹下有哪些插件目录而不显示内部文件。不提供深度参数则默认显示完整文件树。

忽略文件（.workspaceignore）：在工作区根目录放置.workspaceignore文件可排除特定文件和文件夹，语法与.gitignore完全相同。支持通配符（*.log、node_modules/等）、中文路径和文件名。特殊字符（如方括号[]）需要用反斜杠转义，例如要忽略名为[废案]的文件夹，写法为\[废案\]/。

调试模式：在Plugin/WorkspaceInjector/.env文件中设置WORKSPACE_INJECTOR_DEBUG=true开启调试日志，重启VCP服务后每次插件运行时会在控制台打印详细执行步骤，包括接收到的消息内容、找到的占位符、解析的路径等。

可能遇到的问题：
1. 占位符未被替换（原样显示在提示词中）——别名未在config.env的WORKSPACE_ALIASES中配置，或JSON格式错误（路径反斜杠未转义），或指定的路径不存在。
2. 文件树过长导致Token超限——大型项目（如含有node_modules的前端项目）文件树可能有数千行，添加深度限制参数（如::2）或使用.workspaceignore排除大目录。
3. .workspaceignore不生效——npm install ignore依赖未安装，在VCPToolBox根目录运行npm install ignore后重启服务。
4. 中文路径或文件名显示乱码——确认系统编码设置，Windows环境下可能需要设置PowerShell的编码为UTF-8。
5. 别名配置后不生效——config.env修改后需要重启VCP服务才能生效，热加载不适用于config.env的修改。
6. 子目录组合路径解析错误——确认别名对应的根路径末尾没有多余的斜杠，子目录路径使用正斜杠/分隔（跨平台兼容）。

可能的解决办法：
1. 开启WORKSPACE_INJECTOR_DEBUG=true后查看控制台日志，确认插件是否正确识别了占位符和路径。
2. 使用在线JSON验证工具检查WORKSPACE_ALIASES的JSON格式是否合法。
3. 对于大型项目，推荐使用::2或::3的深度限制，配合.workspaceignore排除node_modules/、.git/、dist/等无关目录。
4. 如果需要临时查看某个目录结构，直接使用绝对路径方式（方式二）最为便捷，无需修改配置。
5. 在.workspaceignore中添加常见的大目录：node_modules/、.git/、dist/、build/、__pycache__/、*.pyc等。
