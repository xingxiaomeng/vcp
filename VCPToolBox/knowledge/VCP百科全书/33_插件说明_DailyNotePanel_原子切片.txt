[VCP百科全书] 插件说明——DailyNotePanel（原子切片）

一、插件身份
- name: DailyNotePanelRouter
- displayName: DailyNotePanel 路由胶水插件
- pluginType: service
- command: 无（自动触发）

二、核心能力
- 通过 PluginManager 在主服务上注册 DailyNotePanel 的静态页面端点和 dailynote API 端点。第一阶段使用测试前缀 /DailyNotePanel2 与 /dailynote_api2，验证通过后可切换为正式前缀。
- 以自动注入/后台服务方式提供能力，不要求直接命令调用。

三、占位符与注入点
- systemPromptPlaceholders: 无

四、配置重点
- 优先核对插件目录内的 plugin-manifest.json(.block) 与 config.env.example 必填项。
- 外部API类插件重点检查：密钥、端点、超时、代理与速率限制。
- 关键配置参数:
  * DebugMode: 是否启用调试日志(默认: false)
  * PanelPathPrefix: 页面路由前缀(默认: /AdminPanel/DailyNotePanel)
  * ApiPathPrefix: API路由前缀(默认: /AdminPanel/dailynote_api)

五、排障建议
- 插件无响应：先检查 manifest 启用状态、入口脚本、运行时依赖。
- 结果异常：核对命令名与参数字段，再查看插件目录缓存/日志文件。

六、调用示例
触发示例：启动服务后由路由或事件系统被动触发，无需手动命令。
