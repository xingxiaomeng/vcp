[VCP百科全书] 插件说明——MCPO（原子切片）

一、插件身份
- name: MCPO
- displayName: MCPO 工具桥接器
- pluginType: synchronous
- command: list_tools, call_tool, get_tool_info, manage_server, discover_tools, list_configs, health_check

二、核心能力
- 基于 mcpo 的 MCP 工具桥接插件，能够自动发现、缓存和调用 MCP 工具，支持多种 MCP 服务器类型。
- 提供可调用命令：list_tools、call_tool、get_tool_info、manage_server、discover_tools、list_configs、health_check。

三、占位符与注入点
- systemPromptPlaceholders: 无

四、配置重点
- 优先核对插件目录内的 plugin-manifest.json(.block) 与 config.env.example 必填项。
- 外部API类插件重点检查：密钥、端点、超时、代理与速率限制。

五、排障建议
- 插件无响应：先检查 manifest 启用状态、入口脚本、运行时依赖。
- 结果异常：核对命令名与参数字段，再查看插件目录缓存/日志文件。

六、调用示例

1. list_tools - 列出所有可用工具
<<<[TOOL_REQUEST]>>>
tool_name:「始」MCPO「末」,
action:「始」list_tools「末」
<<<[END_TOOL_REQUEST]>>>

2. call_tool - 调用指定工具
<<<[TOOL_REQUEST]>>>
tool_name:「始」MCPO「末」,
action:「始」call_tool「末」,
tool_name_param:「始」example_tool「末」,
params:「始」{"key": "value"}「末」
<<<[END_TOOL_REQUEST]>>>

3. manage_server - 管理MCP服务器
<<<[TOOL_REQUEST]>>>
tool_name:「始」MCPO「末」,
action:「始」manage_server「末」,
server_action:「始」start「末」
<<<[END_TOOL_REQUEST]>>>
