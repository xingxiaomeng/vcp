[VCP百科全书] VCPEverything / 本地文件秒搜 - Everything HTTP搜索接口

一、插件基本信息
1. 插件名称：VCPEverything / 本地文件秒搜 (Everything)
2. 插件类型：同步(synchronous)
3. 版本号：1.1.0
4. 所属模块：VCPDistributedServer/Plugin
5. 功能一句话总结：通过Everything HTTP服务器实现毫秒级本地文件搜索

二、功能定位
VCPEverything是一个基于Everything搜索引擎的本地文件快速查找插件。它通过HTTP协议与Everything内置的HTTP服务器通信，实现对本地文件系统的毫秒级搜索。

核心能力：
- 利用Everything索引实现毫秒级全文检索
- 支持Everything高级搜索语法（路径过滤、大小过滤、日期过滤等）
- 自动提取搜索结果中的完整文件路径
- 返回结构化的JSON结果，包含查询词、结果数量、文件路径列表
- 使用Node.js原生http模块，无额外依赖
- 支持调试模式，可输出详细日志到stderr

适用场景：快速定位本地文件、批量查找特定类型文件、配合FileOperator进行文件操作前的路径获取。

三、配置参数
配置文件路径：Plugin/VCPEverything/config.env

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| EVERYTHING_PORT | integer | 8025 | Everything HTTP服务器的端口号，必须与Everything设置中Tools -> Options -> HTTP Server配置的端口一致 |
| DEBUG_MODE | boolean | false | 是否开启调试日志，true时会在stderr输出详细请求和响应信息 |

配置示例：
```
EVERYTHING_PORT=8025
DEBUG_MODE=false
```

前置条件：
1. 必须安装Voidtools Everything软件
2. 必须在Everything设置中启用HTTP服务器（Tools -> Options -> HTTP Server -> Enable HTTP Server）
3. HTTP服务器端口必须与EVERYTHING_PORT配置一致

四、调用命令列表

【主命令】search - 搜索本地文件

功能描述：在整台计算机上快速搜索文件，返回匹配的文件路径列表。搜索结果可直接作为FileOperator等插件的输入参数。

参数列表：
| 参数名 | 类型 | 是否必需 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| query | string | 是 | 无 | 搜索关键词，支持Everything高级搜索语法。例如："path:D:\\videos\\ *.mp4 size:>1gb" |
| maxResults | integer | 否 | 100 | 返回的最大结果数量限制 |

完整调用格式：
<<<[TOOL_REQUEST]>>>
tool_name:「始」LocalSearchController「末」,
command:「始」search「末」,
query:「始」*.txt「末」,
maxResults:「始」50「末」
<<<[END_TOOL_REQUEST]>>>

返回值说明：
成功返回格式：
{
  "status": "success",
  "result": {
    "searchQuery": "*.txt",
    "resultCount": 150,
    "results": [
      "C:\\Users\\Name\\Documents\\file1.txt",
      "D:\\Projects\\notes.txt"
    ]
  }
}

失败返回格式：
{
  "status": "error",
  "error": "Connection to Everything HTTP server refused on port 8025. Please ensure Everything is running and the HTTP server is enabled..."
}

六、注意事项与限制
1. 必须预先安装并运行Voidtools Everything软件
2. 必须在Everything设置中启用HTTP服务器功能
3. 默认连接127.0.0.1:8025，只支持本地访问
4. 搜索速度取决于Everything索引构建状态，首次使用建议等待索引完成
5. 超时时间为30000毫秒（30秒）
6. 返回结果为完整绝对路径，可直接用于其他文件操作插件
7. 如果HTTP服务器未启动或端口不匹配，会返回ECONNREFUSED错误
8. 不支持正则表达式搜索（受限于Everything HTTP接口）
9. 大小写不敏感搜索由Everything控制，不在插件层处理

七、关联插件
- FileOperator：可使用VCPEverything返回的路径进行文件读写操作
- LocalSearchController：此为同一插件的不同命名引用
