[VCP百科全书] PTYShellExecutor / PTY Shell 执行器 - Linux本地命令执行

一、插件基本信息
1. 插件名称：PTYShellExecutor / PTY Shell 执行器
2. 插件类型：混合服务(hybridservice)
3. 版本号：1.0.0
4. 所属模块：VCPDistributedServer/Plugin
5. 功能一句话总结：在Linux桌面环境下使用PTY会话执行本地Shell命令

二、功能定位
PTYShellExecutor是一个强大的Linux本地Shell命令执行插件，使用node-pty创建持久化的PTY(伪终端)会话。支持bash/fish/zsh等多种Shell，保持环境变量和会话状态。核心特性包括：
- 同步执行模式：在PTY会话中执行命令，保持环境变量和会话状态
- 异步任务模式：后台执行长时间任务(如codex编译)，立即返回taskId供后续查询
- 任务管理：支持查询、取消、列出所有异步任务
- 自动过滤ANSI转义序列和Shell Integration标记，返回干净的文本输出

该插件专为Linux桌面环境设计，适用于需要持久化Shell会话、环境状态保持的场景。

三、配置参数
配置文件路径：Plugin/PTYShellExecutor/config.env

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| SHELL_RETURN_MODE | string | delta | 返回模式: delta(仅返回增量输出)或full(返回完整终端内容) |
| SHELL_PRIORITY | string | bash,fish,zsh | 默认Shell优先级，逗号分隔，按顺序检测可用Shell |
| FORBIDDEN_COMMANDS | string | rm -rf /,mkfs,dd if= | 禁止执行的命令关键字，逗号分隔 |
| AUTH_REQUIRED_COMMANDS | string | sudo,su,chmod 777 | 需要确认的命令关键字，逗号分隔 |
| COMMAND_TIMEOUT | integer | 60000 | 命令执行超时时间(毫秒)，默认60秒 |

配置示例:
SHELL_RETURN_MODE=delta
SHELL_PRIORITY=fish,zsh,bash
FORBIDDEN_COMMANDS=rm -rf /,mkfs,dd if=/dev/zero
AUTH_REQUIRED_COMMANDS=sudo,su,chmod 777
COMMAND_TIMEOUT=120000

四、调用命令列表

【单命令模式】

命令名称：PTYShellExecutor
功能描述：在Linux桌面环境下执行本地Shell命令，支持同步执行和异步任务模式

参数列表:
| 参数名 | 类型 | 必需 | 默认值 | 说明 |
|--------|------|------|--------|------|
| action | string | 否 | execute | 操作类型: execute(同步执行)/async(异步执行)/query(查询任务)/cancel(取消任务)/list(列出任务) |
| command | string | 条件 | - | 要执行的Shell命令(execute/async模式必需) |
| taskId | string | 条件 | - | 异步任务ID(query/cancel模式必需) |
| shell | string | 否 | - | 指定Shell类型: fish/zsh/bash |
| newSession | boolean | 否 | false | 是否创建新的PTY会话 |
| cwd | string | 否 | - | 工作目录(async模式可用) |

返回值说明:
- execute模式: 返回命令执行结果，包含stdout/stderr/exitCode
- async模式: 返回{taskId: "xxx", status: "running"}
- query模式: 返回任务的当前状态和输出
- cancel模式: 返回取消结果
- list模式: 返回所有任务列表

六、注意事项与限制
1. 仅支持Linux桌面环境，Windows/macOS不适用
2. 必须安装node-pty依赖：npm install node-pty
3. 系统必须安装bash/fish/zsh中的至少一种Shell
4. 危险命令(rm -rf /等)会被拦截，可在FORBIDDEN_COMMANDS中配置
5. 特权命令(sudo/su等)需要确认，可在AUTH_REQUIRED_COMMANDS中配置
6. PTY会话会保持环境变量，但进程退出后状态不保留
7. 异步任务的输出会持续累积，需定期查询获取最新输出
8. 超时会强制终止命令执行，长任务建议使用async模式
9. 通信协议为direct(直接调用)，非stdio
10. 最大超时时间为600000毫秒(10分钟)
