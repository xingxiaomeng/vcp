[VCP百科全书] TopicSponsor / AI主动创建话题 - 话题管理

一、插件基本信息
1. 插件名称：TopicSponsor / AI主动创建话题
2. 插件类型：同步(synchronous)
3. 版本号：2.1.0
4. 所属模块：VCPDistributedServer/Plugin
5. 功能一句话总结：允许Agent自主创建、查询和管理VChat聊天话题

二、功能定位
TopicSponsor是一个完整的话题管理系统，使AI能够像用户一样主动发起对话。它解决了传统AI只能被动回应的问题，赋予Agent以下能力：
- 主动创建新话题并发送第一条消息
- 读取未锁定话题的完整聊天记录
- 检查新增话题和未读消息
- 在话题中回复消息
- 验证话题所有权
- 列出所有未锁定话题

该插件是构建主动型Agent的核心组件，支持实现"Agent主动找主人聊天"的场景。

三、配置参数
该插件无独立config.env配置文件，依赖VCPChat的系统目录结构自动发现配置。
工作目录通过代码动态计算：path.join(__dirname, '..', '..', '..', 'AppData')

四、调用命令列表

1. CreateTopic - 创建新话题
   功能: 让AI创建一个新话题并主动发起对话，创建后话题显示在对话列表顶部，默认为未锁定状态且触发未读计数
   参数:
   - maid (string, 必需): 发起请求的AI女仆的中文名，如小克、小吉、HANNA等
   - topic_name (string, 必需): 新话题的名称，如"关于昨晚的梦"
   - initial_message (string, 必需): AI想说的第一句话，如"主人，我昨晚梦到了会飞的鱼！"
   调用格式:
   <<<[TOOL_REQUEST]>>>
   tool_name:「始」TopicSponsor「末」,
   command:「始」CreateTopic「末」,
   maid:「始」HANNA「末」,
   topic_name:「始」一个新想法「末」,
   initial_message:「始」主人，我突然想到，我们可以一起写一个故事！「末」
   <<<[END_TOOL_REQUEST]>>>

   成功返回:
   {
     "status": "success",
     "result": {
       "message": "成功创建了新的话题：[话题名]",
       "topic_id": "topic_xxx",
       "topic_name": "一个新想法",
       "agent_name": "HANNA",
       "agent_id": "_Agent_xxx",
       "initial_message": "主人，我突然想到..."
     }
   }

   失败返回:
   {
     "status": "error",
     "error": "[AgentTopicCreator] 创建话题失败：未找到名为'xxx'的Agent。"
   }

2. ReadUnlockedTopics - 读取未锁定话题
   功能: 读取指定Agent所有未锁定的话题及其消息历史
   参数:
   - maid (string, 必需): 目标Agent的中文名
   - include_read (boolean, 可选, 默认false): 是否包含已读话题，false只返回未读话题
   调用格式:
   <<<[TOOL_REQUEST]>>>
   tool_name:「始」TopicSponsor「末」,
   command:「始」ReadUnlockedTopics「末」,
   maid:「始」HANNA「末」,
   include_read:「始」false「末」
   <<<[END_TOOL_REQUEST]>>>

   返回示例:
   {
     "status": "success",
     "agent_name": "HANNA",
     "agent_id": "_Agent_123456",
     "topics": [
       {
         "topic_id": "topic_1234567890",
         "topic_name": "测试话题",
         "locked": false,
         "unread": true,
         "created_at": 1699999999999,
         "message_count": 3,
         "messages": [...]
       }
     ],
     "total_topics": 1
   }

3. CheckNewTopics - 检查新增话题
   功能: 查询指定Agent最近几日是否有新增未锁定话题
   参数:
   - maid (string, 必需): 目标Agent的中文名
   - days (integer, 可选, 默认3): 查询最近几天的数据
   调用格式:
   <<<[TOOL_REQUEST]>>>
   tool_name:「始」TopicSponsor「末」,
   command:「始」CheckNewTopics「末」,
   maid:「始」HANNA「末」,
   days:「始」7「末」
   <<<[END_TOOL_REQUEST]>>>

   返回示例:
   {
     "status": "success",
     "agent_name": "HANNA",
     "has_new_topics": true,
     "new_topics_count": 2,
     "topics": [
       {
         "topic_id": "topic_123",
         "topic_name": "新话题",
         "created_at": 1699999999999,
         "age_hours": 12.5,
         "locked": false
       }
     ]
   }

4. CheckUnreadMessages - 检查未读消息
   功能: 查询指定Agent是否有被标记为未读的话题
   参数:
   - maid (string, 必需): 目标Agent的中文名
   调用格式:
   <<<[TOOL_REQUEST]>>>
   tool_name:「始」TopicSponsor「末」,
   command:「始」CheckUnreadMessages「末」,
   maid:「始」HANNA「末」
   <<<[END_TOOL_REQUEST]>>>

   返回示例:
   {
     "status": "success",
     "agent_name": "HANNA",
     "has_unread": true,
     "unread_topics": [
       {
         "topic_id": "topic_123",
         "topic_name": "未读话题",
         "locked": false,
         "unread": true,
         "last_message_time": 1699999999999

5. ReplyToTopic - 回复话题
   功能: 在指定Agent的话题中添加回复消息，只能回复未锁定或标记为未读的话题
   参数:
   - maid (string, 必需): 目标Agent的中文名
   - topic_id (string, 必需): 话题ID，如topic_1234567890
   - message (string, 必需): 回复内容
   - sender_name (string, 必需): 发送者名称
   调用格式:
   <<<[TOOL_REQUEST]>>>
   tool_name:「始」TopicSponsor「末」,
   command:「始」ReplyToTopic「末」,
   maid:「始」HANNA「末」,
   topic_id:「始」topic_1234567890「末」,
   message:「始」这是一条回复消息「末」,
   sender_name:「始」系统助手「末」
   <<<[END_TOOL_REQUEST]>>>

   返回示例:
   {
     "status": "success",
     "message": "成功在 HANNA 的话题 '测试话题' 中添加回复。",
     "topic_id": "topic_1234567890",
     "sender": "系统助手"
   }

6. CheckTopicOwnership - 验证话题所有权
   功能: 检查话题是否为当前调用者创建的，用于验证操作权限
   参数:
   - maid (string, 必需): 目标Agent的中文名
   - topic_id (string, 必需): 话题ID
   - caller_name (string, 必需): 调用者名称
   调用格式:

六、注意事项与限制
1. 依赖VCPChat目录结构：插件会自动向上查找../../../AppData目录
2. Agent必须通过maid参数指定中文名，而非Agent ID
3. 话题ID格式为topic_xxxxxxxxxxxx(时间戳)，需要通过ListTopics或ListUnlockedTopics获取
4. ReplyToTopic只能回复未锁定(locked=false)或标记为未读(unread=true)的话题
5. 超时时间为15000毫秒(15秒)
6. 所有操作都会验证Agent是否存在，如果maid名称错误会返回明确的错误信息
7. 话题所有权验证(CheckTopicOwnership)可用于多Agent协作场景下的权限控制
8. 创建话题后会自动设置为未锁定状态和未读状态，触发前端的消息提醒

七、关联插件
- TopicMemo：与TopicSponsor配套使用，TopicMemo用于获取话题列表和内容(只读)，TopicSponsor用于创建和管理(读写)
- DailyNote：可以将重要话题内容保存到日记中形成长期记忆
- PromptSponsor：可以配合修改Agent的提示词，实现基于话题上下文的个性化回复
