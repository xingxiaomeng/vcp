一、main.js - Electron主进程核心

1.1 文件定位
路径：VCPChat/main.js
大小：约61KB，1400+行
角色：Electron应用入口，主进程唯一入口点

1.2 核心职责

应用生命周期管理：
├── app.whenReady()      # 应用就绪事件
├── app.on('window-all-closed')  # 所有窗口关闭
├── app.on('activate')   # 应用激活（macOS Dock点击）
├── app.on('before-quit') # 退出前清理
└── 单实例锁机制        # 防止多开

窗口管理：
├── createWindow()       # 创建主窗口
├── mainWindow管理      # BrowserWindow实例
├── 窗口状态恢复        # 位置、大小、最大化状态
├── 托盘图标管理        # Tray + ContextMenu
└── 启动画面控制        # splash.html

系统级功能：
├── 全局快捷键注册     # Ctrl+Alt+V等
├── 剪贴板监控         # 图片自动粘贴
├── 协议处理           # vcp:// URL Scheme
├── 自动更新检查       # 版本比对
└── 系统通知           # 原生通知

子进程管理：
├── VCPDistributedServer启动
├── 音频引擎(WASAPI)启动
├── Python子进程执行
└── Worker线程管理

1.4 IPC事件注册

main.js中注册的主要IPC通道：
├── 设置管理：load-settings, save-settings
├── Agent管理：get-agents, save-agent-config, create-agent
├── 文件操作：select-file, select-directory, save-file
├── 聊天功能：send-message, load-chat-history
├── 窗口控制：minimize-window, maximize-window, close-window
└── 系统功能：open-external, show-in-folder

1.5 安全设计

上下文隔离 (contextIsolation: true)：
- 渲染进程无法直接访问Node.js API
- 预加载脚本在独立上下文中运行

沙箱 (sandbox: true)：
- 渲染进程运行在受限环境
- 限制系统级操作

IPC白名单：
- 只有显式注册的通道可用
- 防止未授权的系统访问

二、preload.js - 安全桥接层

2.1 文件定位
路径：VCPChat/preload.js
大小：约25KB，250+行
角色：渲染进程与主进程之间的安全桥梁

2.2 核心职责

contextBridge API暴露：
├── 安全地暴露受控API给渲染进程
├── 防止渲染进程直接访问electron/Node API
└── 实现最小权限原则

IPC通道白名单：
├── invoke通道：需要返回值的调用
├── send通道：单向消息发送
├── on通道：事件监听
└── 所有通道需显式定义

2.3 关键API暴露

```javascript
// 第58-120行：electronAPI主对象
contextBridge.exposeInMainWorld('electronAPI', {
    // 设置
    loadSettings: () => ipcRenderer.invoke('load-settings'),
    saveSettings: (settings) => ipcRenderer.invoke('save-settings', settings),

    // Agent管理
    getAgents: () => ipcRenderer.invoke('get-agents'),
    getAgentConfig: (agentId) => ipcRenderer.invoke('get-agent-config', agentId),
    saveAgentConfig: (agentId, config) => ipcRenderer.invoke('save-agent-config', agentId, config),
    createAgent: (agentName, initialConfig) => ipcRenderer.invoke('create-agent', agentName, initialConfig),
    deleteAgent: (agentId) => ipcRenderer.invoke('delete-agent', agentId),

    // 话题管理
    getAgentTopics: (agentId) => ipcRenderer.invoke('get-agent-topics', agentId),
    createNewTopicForAgent: (agentId, topicName) => ipcRenderer.invoke('create-new-topic-for-agent', agentId, topicName),
    deleteTopic: (agentId, topicId) => ipcRenderer.invoke('delete-topic', agentId, topicId),

    // 文件操作
    selectDirectory: () => ipcRenderer.invoke('select-directory'),
    selectFile: () => ipcRenderer.invoke('select-file'),

    // 聊天
    onReceiveMessage: (callback) => ipcRenderer.on('receive-message', callback),
    sendMessage: (data) => ipcRenderer.send('send-message', data),

    // 音乐控制
    musicPlay: () => ipcRenderer.invoke('music-play'),
    musicPause: () => ipcRenderer.invoke('music-pause'),
    musicLoad: (filePath) => ipcRenderer.invoke('music-load', filePath),
});
```

2.4 安全机制详解

通道验证：
```javascript
// 第22-44行：invoke通道白名单
let validChannels = [
    'music-load', 'music-play', 'music-pause',
    'music-get-devices', 'music-set-eq', ...
];
if (validChannels.includes(channel)) {
    return ipcRenderer.invoke(channel, data);
}
```

事件剥离：
```javascript
// 第52行：剥离event对象（包含sender）
ipcRenderer.on(channel, (event, ...args) => func(...args));
```

2.5 为什么需要preload.js？

安全隔离：
- 渲染进程运行在Web环境，不应有系统访问权限
- preload.js是唯一可以访问Node.js API的地方

受控暴露：
- 开发者显式决定哪些功能对渲染进程可用
- 防止恶意脚本利用Electron API

架构清晰：
- 分离渲染逻辑和系统访问
- 便于维护和审计

    // 构建消息对象

async function loadAgents() {

        const agents = await window.electronAPI.getAgents();

        renderAgentList(agents);

五、三文件协作流程

5.1 应用启动流程

1. Electron启动
   └── 加载main.js（主进程）

2. main.js执行
   ├── 创建BrowserWindow
   ├── 加载main.html
   ├── 注入preload.js
   └── 注册IPC处理器

3. 渲染进程初始化
   ├── 加载main.html
   ├── 执行preload.js（建立IPC桥接）
   └── 执行renderer.js

4. renderer.js初始化
   ├── 获取DOM元素
   ├── 绑定事件监听
   ├── 加载初始数据（通过IPC）
   └── 建立WebSocket连接

5.2 消息发送流程

    ↓

    ↓
组装消息对象
    ↓
本地渲染（ optimistic UI）
    ↓
webSocketClient.send() （renderer.js）
    ↓
WebSocket传输
    ↓

    ↓
AI模型API调用
    ↓
响应返回
    ↓

    ↓

5.3 系统功能调用流程

用户点击"打开文件"
    ↓

    ↓

    ↓

    ↓

    ↓

    ↓

六、开发注意事项

6.1 main.js开发注意

✓ 所有系统API调用必须在此完成
✓ 错误处理必须完善，防止主进程崩溃
✓ IPC处理器需要验证输入参数
✓ 资源清理在before-quit中完成

6.2 preload.js开发注意

✓ 只暴露必要API，遵循最小权限原则
✓ IPC通道白名单需要显式维护
✓ 敏感操作需要二次确认
✓ 不要暴露ipcRenderer原始对象

6.3 renderer.js开发注意

✗ 永远不要直接require('electron')或Node模块
✓ 所有系统访问通过window.electronAPI
✓ 大数据处理使用Worker线程
✓ 内存泄漏防护（清理事件监听）
