

**国服 SiliconFlow IndexTTS-2 完整实用文档（2026最新）**

**核心模型说明**：
- **上传参考音频** 时使用：`"IndexTeam/IndexTTS-2"`
- **生成语音（Create Speech）** 时使用：`"IndexTeam/IndexTTS-2"`（或直接使用平台显示的 IndexTTS-2 模型名称，以实际控制台/文档为准。目前官方推荐使用完整名称 `IndexTeam/IndexTTS-2`）

**重要**：`voice`（预设或上传的 uri）与 `references` **互斥**，只能二选一。

### 1. 请求方法（Create Speech - 生成语音）

**Endpoint**：`POST https://api.siliconflow.cn/v1/audio/speech`  
**认证**：`Authorization: Bearer sk-你的API Key`  
**Content-Type**：`application/json`

**主要请求参数**（JSON Body）：

| 参数              | 类型          | 必填 | 默认值     | 说明 |
|-------------------|---------------|------|------------|------|
| model            | string       | 是   | -         | `"IndexTeam/IndexTTS-2"` |
| input            | string       | 是   | -         | 要转成语音的文本（支持 `[S1]` `[S2]` 多角色标签） |
| voice            | string       | 否   | -         | 预设音色或上传后返回的 uri（与 references 互斥） |
| references       | array[object]| 否   | -         | 动态参考音频数组（与 voice 互斥，支持多个文件） |
| response_format  | string       | 否   | mp3       | mp3 / opus / wav / pcm |
| sample_rate      | number       | 否   | 32000     | 采样率 |
| speed            | float        | 否   | 1.0       | 语速（0.25 ~ 4.0） |
| gain             | float        | 否   | 0         | 增益（-10 ~ 10） |
| stream           | boolean      | 否   | true      | 是否流式 |

**curl 示例**（使用默认音色）：
```bash
curl --request POST \
  --url https://api.siliconflow.cn/v1/audio/speech \
  --header 'Authorization: Bearer sk-你的key' \
  --header 'Content-Type: application/json' \
  --data '{
    "model": "IndexTeam/IndexTTS-2",
    "input": "你好，这是一段测试文本。",
    "voice": "IndexTeam/IndexTTS-2:alex",
    "response_format": "mp3",
    "speed": 1.0
  }'
```

**Python 获取音频示例**：
```python
import requests

url = "https://api.siliconflow.cn/v1/audio/speech"
headers = {
    "Authorization": "Bearer sk-你的key",
    "Content-Type": "application/json"
}
data = {
    "model": "IndexTeam/IndexTTS-2",
    "input": "测试文本",
    "voice": "IndexTeam/IndexTTS-2:alex",
    "response_format": "mp3"
}

response = requests.post(url, json=data, headers=headers)
if response.status_code == 200:
    with open("output.mp3", "wb") as f:
        f.write(response.content)
    print("音频已保存为 output.mp3")
else:
    print("错误:", response.text)
```

**获取方法**：响应是**二进制音频数据**，直接将 `response.content` 写入文件（.mp3/.wav 等）。支持流式播放（stream=true）。

### 3. 创建参考音频方法（预上传，推荐长期使用）

**Endpoint**：`POST https://api.siliconflow.cn/v1/uploads/audio/voice`  
**请求方式**：`multipart/form-data`

**参数**（Form 表单）：
- `model`：`"IndexTeam/IndexTTS-2"`（必填）
- `customName`：自定义名称（如 `my_own_voice`、`xiaoming_voice`）
- `text`：参考音频对应的**精确文字内容**（必须高度匹配）
- `file`：音频文件（推荐 8–15 秒，清晰、单人、无噪音）

**curl 示例**：
```bash
curl -X POST "https://api.siliconflow.cn/v1/uploads/audio/voice" \
  -H "Authorization: Bearer sk-你的key" \
  -F "model=IndexTeam/IndexTTS-2" \
  -F "customName=my_own_voice" \
  -F "text=你好，我是你的专属参考音频，这里是一段测试文字内容。" \
  -F "file=@你的参考音频.mp3"
```

**返回示例**：
```json
{
  "uri": "speech:my_own_voice:xxx:xxx"
}
```

保存返回的 `uri`，后续生成时使用。

### 4. 使用参考音频方法（已上传的 uri）

在 speech 请求中使用返回的 uri：
```json
{
  "model": "IndexTeam/IndexTTS-2",
  "input": "今天天气真不错。",
  "voice": "speech:my_own_voice:xxx:xxx",   // 填上传返回的 uri
  "response_format": "mp3"
}
```

### 5. 使用多个文件创建/使用方法（动态 references）

不预上传，直接在生成请求中传入多个参考音频（效果通常更好）。

**请求体示例**：
```json
{
  "model": "IndexTeam/IndexTTS-2",
  "input": "[S1]你好，今天怎么样？[S2]我感觉很不错。",
  "references": [
    {
      "audio": "https://your-domain.com/ref1.mp3",   // 支持公开URL或base64
      "text": "第一个参考音频对应的完整文字内容"
    },
    {
      "audio": "https://your-domain.com/ref2.wav",
      "text": "第二个参考音频对应的完整文字内容"
    }
  ],
  "response_format": "wav"
}
```

**说明**：
- `references` 是数组，可传入多个对象。
- 此时**不要**传入 `voice` 字段。
- 适合同一个说话人提供不同情绪/场景样本，或多角色对话。

**使用建议**：
- 简单单音色长期使用 → 先上传得到 uri → 用 `voice`。
- 需要更高克隆品质 → 用 `references` 传入多个文件。
- 参考音频质量最重要（清晰、单人、无噪音最佳）。

这个版本已经全部统一使用 **IndexTeam/IndexTTS-2**。  
实际使用时，建议登录国服控制台（https://cloud.siliconflow.cn/models）查看 IndexTTS-2 的最新模型名称，以防有细微调整。

### 1. 获取我的参考音频列表（Retrieve Voice List）

**Endpoint**：`GET https://api.siliconflow.cn/v1/audio/voice/list`  
**方法**：GET  
**认证**：`Authorization: Bearer sk-你的API Key`

**curl 示例**：
```bash
curl --request GET \
  --url https://api.siliconflow.cn/v1/audio/voice/list \
  --header 'Authorization: Bearer sk-你的key'
```

**Python 示例**：
```python
import requests

url = "https://api.siliconflow.cn/v1/audio/voice/list"
headers = {
    "Authorization": "Bearer sk-你的key"
}

response = requests.get(url, headers=headers)
if response.status_code == 200:
    data = response.json()
    print("你的参考音频列表：")
    for voice in data.get("results", []):
        print(f"customName: {voice.get('customName')}")
        print(f"uri: {voice.get('uri')}")
        print(f"model: {voice.get('model')}")
        print(f"text: {voice.get('text')[:100]}...")  # 显示部分文本
        print("-" * 50)
else:
    print("错误:", response.status_code, response.text)
```

**返回示例**（简化）：
```json
{
  "results": [
    {
      "uri": "speech:my_own_voice:xxx:xxx",
      "customName": "my_own_voice",
      "model": "IndexTeam/IndexTTS-2",
      "text": "你好，我是你的专属参考音频..."
    },
    {
      "uri": "speech:xiaoming_voice:yyy:yyy",
      "customName": "xiaoming_voice",
      ...
    }
  ]
}
```

你可以通过 `customName` 或 `uri` 来识别和管理自己的参考音频。

### 2. 删除参考音频（Delete Voice）

**Endpoint**：`POST https://api.siliconflow.cn/v1/audio/voice/deletions`  
**方法**：POST  
**认证**：`Authorization: Bearer sk-你的API Key`  
**Content-Type**：`application/json`

**请求参数**（JSON Body）：

| 参数     | 类型   | 必填 | 说明 |
|----------|--------|------|------|
| uri     | string | 是   | 要删除的参考音频的 uri（从列表中获取） |

**curl 示例**：
```bash
curl --request POST \
  --url https://api.siliconflow.cn/v1/audio/voice/deletions \
  --header 'Authorization: Bearer sk-你的key' \
  --header 'Content-Type: application/json' \
  --data '{
    "uri": "speech:my_own_voice:xxx:xxx"
  }'
```

**Python 示例**：
```python
import requests

url = "https://api.siliconflow.cn/v1/audio/voice/deletions"
headers = {
    "Authorization": "Bearer sk-你的key",
    "Content-Type": "application/json"
}
data = {
    "uri": "speech:my_own_voice:xxx:xxx"   # 从列表中复制 uri
}

response = requests.post(url, json=data, headers=headers)
if response.status_code == 200:
    print("删除成功！")
else:
    print("删除失败:", response.status_code, response.text)
```

**注意事项**：
- 删除后该 `uri` 无法再用于 TTS 生成。
- 删除是**永久性**操作，无法恢复。请先通过列表确认要删除的 uri。
- 你可以一次性只删除一个；如果需要删除多个，可循环调用删除接口。
- 删除不会影响平台默认音色（alex、anna 等），只删除你自己上传的自定义参考音频。
- 建议在删除前先调用列表接口，确认 `customName` 和 `uri` 对应正确。

**推荐流程**：
1. 先调用 **获取列表** 接口，查看所有自定义参考音频。
2. 确认要删除的 `uri`。
3. 调用 **删除** 接口。
